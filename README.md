# Cordial

> Cordially offering the prepared cordial.
>
> 热情递出那杯早已调好的甜果汁

<br />

## Cordial是什么？

Cordial是一个网站项目，旨在向用户特殊化甚至随机化地提供站主个人**事先准备好的文本等目标内容**

<br />

## 具体介绍

站主为每份**目标内容**指定（或不指定）其分发方式，包括确定的接收者或潜在的接收者、确切的时间或模糊的时间段、对其它**目标内容**是否已分发的依赖程度等

决策内核综合**目标内容**、用户访问和现实状况信息做出决策，向某个或一些用户推送分发某份或多份**目标内容**

Cordial一般部署为一个公开的网络站点，任何人都可访问、注册并登录账号；站主需要为**内容**中提及的或希望指定接收的用户事先创建好账户

如上所述，虽然Cordial有像书籍一样的单向信息传递性，但用户无法直接获取站主为其准备的所有**目标内容**，而更像是在逐封接收书信，Cordial的决策依托于站主的分发要求，对用户来说充满神秘性，站主也可通过在要求中留存模糊空间来一定程度上减少对分发行为的掌控，一方面提高了配置目标内容分发要求的效率（只关注大致目标），另一方面增强了站主在书写目标内容时的热情

Cordial不建议创建和使用完全公开、无条件即时分发的**目标内容**，而更多鼓励对特定接收者的飘忽不定的延迟书信；Cordial是一个中间人，可以将所有感性的、情绪的、想说的、难言的、真挚的、热情的这样一杯**甜果汁**交给它，由它来晃着杯子决定在遥远的哪一天将**果汁**代为递出

注意：过大的模糊空间可能会使分发完全脱离掌控（如部分内容可能错误地永远不分发给任何人看见）

完整的Cordial应包含以下组件（功能）：

* 加密数据库以保护所有目标内容

* 特殊的数据结构以存放各种目标内容附属信息（分发要求）

* 账号管理系统

* 人工智能体或情境分析算法或一般的逻辑判断算法，构成决策内核，用于做出分发内容的最终决策

* 网站前后端架构
